```
>> id

```
Even if I write the command in lowercase, it is returned in uppercase.

After some readings, I found that `$0` refers to the shell. I tried it in my machine.
```
└─$ echo $0
/usr/bin/zsh
```

```
┌──(kali㉿kali)-[~/Desktop/Bandit/bandit31]
└─$ $0                           
┌──(kali㉿kali)-[~/Desktop/Bandit/bandit31]
└─$ 
```
Entering `$0` in the shell, I get a shell. Doing the same in the bandit32 shell.
```
>> $0
$ id
uid=11033(bandit33) gid=11032(bandit32) groups=11032(bandit32)
$ whoami
bandit33
```
