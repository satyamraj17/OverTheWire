# Level Goal

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL/TLS and which don’t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

Helpful note: Getting “DONE”, “RENEGOTIATING” or “KEYUPDATE”? Read the “CONNECTED COMMANDS” section in the manpage.

# Solution
With a range of 1000 ports, manually doing it will take a long time. We use `nmap` to find the open ports first. And then we check which port requires SSL/TLS for the connection.
```
bandit16@bandit:~$ nmap -p31000-32000 localhost 
Starting Nmap 7.94SVN ( https://nmap.org ) at 2026-06-04 06:17 UTC
Nmap scan report for localhost (127.0.0.1)
Host is up (0.00012s latency).
Not shown: 996 closed tcp ports (conn-refused)
PORT      STATE SERVICE
31046/tcp open  unknown
31518/tcp open  unknown
31691/tcp open  unknown
31790/tcp open  unknown
31960/tcp open  unknown
```
Since we have only 5 open ports, we can manually check which require SSL/TLS. (The service detection was taking a lot of time)
```
bandit16@bandit:~$ nc localhost 31046
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
^C

bandit16@bandit:~$ nc localhost 31518
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx

bandit16@bandit:~$ nc localhost 31691
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
^C

bandit16@bandit:~$ nc localhost 31790
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx

bandit16@bandit:~$ nc localhost 31960
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
^C
```
3 out of the 5 ports are returning what we are sending. For the remaining 2 (31518, 31790), the connection closes after we send the data. These 2 are using SSL/TLS. Of these 2, one will return the password.
```
bandit16@bandit:~$ openssl s_client -connect localhost:31790 -quiet
Can't use SSL_get_servername
depth=0 CN = SnakeOil
verify error:num=18:self-signed certificate
verify return:1
depth=0 CN = SnakeOil
verify return:1
kSkvUpMQ7lBYyCM4GBPvCvT1BfWRy0Dx
Correct!
(RSA KEY)
```
The command we used in the previous task didn't work. I have to add `-quite` for it to return the private key. Copy this private key, change the permissions and then log in to SSH with it.
