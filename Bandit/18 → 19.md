# Level Goal

The password for the next level is stored in a file readme in the homedirectory. Unfortunately, someone has modified .bashrc to log you out when you log in with SSH.

# Solution
The .bashrc file has been modified, which is why we are unable to log in to the user via SSH. The .bashrc file is a Bash shell script that runs whenever Bash is started interactively. It initialises an interactive shell session. So we have to log in such that it doesn't load anything from .bashrc
```
└─$ ssh bandit18@bandit.labs.overthewire.org -p 2220 bash --norc            
                         _                     _ _ _   
                        | |__   __ _ _ __   __| (_) |_ 
                        | '_ \ / _` | '_ \ / _` | | __|
                        | |_) | (_| | | | | (_| | | |_ 
                        |_.__/ \__,_|_| |_|\__,_|_|\__|
                                                       

                      This is an OverTheWire game server. 
            More information on http://www.overthewire.org/wargames

backend: gibson-0
bandit18@bandit.labs.overthewire.org's password: 
ls
readme
```
Now we can access the .bashrc file and see what was preventing the SSH connection.
```
echo 'Byebye !'
exit 0
```
As soon as we log in, the shell is initialised, and it exits at the end.
