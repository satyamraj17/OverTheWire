# Level Goal

The password for the next level is stored in the file data.txt, which is a hexdump of a file that has been repeatedly compressed. For this level it may be useful to create a directory under /tmp in which you can work. Use mkdir with a hard to guess directory name. Or better, use the command “mktemp -d”. Then copy the datafile using cp, and rename it using mv (read the manpages!)

# Solution
As suggested, first copy the file to a directory in `tmp` directory.
```
bandit12@bandit:~$ mktemp -d
/tmp/tmp.bnSF39WX49
bandit12@bandit:~$ cp data.txt /tmp/tmp.bnSF39WX49
```

Then we reverse the hexdump using `xxd` using the `-r` flag. And use the `file` command to determine the file type, then rename the file with the appropriate extension.
```
bandit12@bandit:/tmp/tmp.bnSF39WX49$ xxd -r data.txt compressed_data

bandit12@bandit:/tmp/tmp.bnSF39WX49$ file compressed_data 
compressed_data: gzip compressed data, was "data2.bin", last modified: Fri Apr  3 15:17:36 2026, max compression, from Unix, original size modulo 2^32 576

bandit12@bandit:/tmp/tmp.bnSF39WX49$ mv compressed_data compressed_data.gz

bandit12@bandit:/tmp/tmp.bnSF39WX49$ gzip -d compressed_data.gz 

bandit12@bandit:/tmp/tmp.bnSF39WX49$ file compressed_data 
compressed_data: bzip2 compressed data, block size = 900k

bandit12@bandit:/tmp/tmp.bnSF39WX49$ mv compressed_data compressed_data.bz2

bandit12@bandit:/tmp/tmp.bnSF39WX49$ file compressed_data
```
